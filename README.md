# TrainConsistManagement

Train Consist Management System
📌 Project Overview

The Train Consist Management System is a Java-based application designed to simulate and manage train bogies efficiently. The project demonstrates core programming concepts such as arrays, collections, stream processing, exception handling, and algorithms, along with proper Git branching strategy and unit testing.

🚀 Features (Use Cases)
🔹 Basic Operations (UC1–UC5)
UC1: Initialize and display bogie list
UC2: Search bogie using linear search
UC3: Add new bogie
UC4: Remove bogie
UC5: Update bogie
🔹 Collections Framework (UC6–UC7)
UC6: Manage bogies using ArrayList
UC7: Ensure unique bogies using HashSet
🔹 Stream API (UC8–UC10)
UC8: Filter bogies based on capacity
UC9: Group bogies using groupingBy()
UC10: Calculate total capacity using reduce()
🔹 Validation (UC11–UC12)
UC11: Regex validation for Train ID and Cargo Code
UC12: Safety validation using allMatch()
🔹 Advanced Concepts (UC13–UC15)
UC13: Performance comparison (Loop vs Stream)
UC14: Custom Exception for invalid capacity
UC15: Runtime exception handling using try-catch-finally
🔹 Algorithms (UC16–UC19)
UC16: Bubble Sort (manual sorting)
UC17: Built-in sorting using Arrays.sort()
UC18: Linear Search
UC19: Binary Search
🔹 Robust Handling (UC20)
UC20: Fail-fast validation during search
🧪 Testing
Unit testing implemented using JUnit
Each use case includes multiple test cases
Ensures correctness, edge case handling, and reliability

🌿 Git Workflow

The project follows a structured branching strategy:

main → Final stable version
develop → Integration branch
feature/UCx → Individual use case implementation

Workflow:

Create feature branch from develop
Implement use case
Commit and push
Merge into develop
Final merge into main
▶️ How to Run
Clone the repository
Open in any Java IDE (VS Code / IntelliJ)
Run any UseCaseX.java file
Run test files inside src/test/java
